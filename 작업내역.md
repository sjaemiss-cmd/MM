# MM 중간고사 학습앱 작업내역

> 작업일: 2026-04-14
> 배포 URL: https://sjaemiss-cmd.github.io/MM/mm-study.html

---

## 1. 프로젝트 개요

마케팅관리(류상희 교수) 중간고사(4/16) 대비를 위한 모바일 퍼스트 학습앱 제작.
강의노트 PDF 5개(Topic 1~5)를 파싱하여 문제은행을 구축하고, 시험 형식을 반영한 퀴즈 앱으로 구현.

---

## 2. 작업 순서

### Step 1: 강의노트 PDF 파싱
- PyMuPDF(`fitz`)로 5개 PDF에서 텍스트 추출
- Topic 1~5, 총 161페이지 분량

| PDF | 페이지 |
|-----|--------|
| Topic 1. 마케팅에 대한 이해 | 32p |
| Topic 2. 시장의 이해 | 26p |
| Topic 3. 세분화, 표적화, 위치화 전략 | 30p |
| Topic 4. 소비자의 이해 | 41p |
| Topic 5. 마케팅 조사 | 32p |

### Step 2: 시험 형식 분석
- `mm 중간고사.txt` 파일 기반
- 54~55문제, 선다형 85% + 주관식 15%
- 핵심: 빈칸, 주어-서술 매칭, 전략 적용 상황, 사례의 포인트

### Step 3: 문제은행 생성 (병렬 에이전트)
- 5개 토픽별 에이전트를 동시 실행하여 문제 생성
- 각 에이전트가 해당 토픽의 PDF 텍스트를 읽고 문제 JSON 생성

| 토픽 | 초기 문제수 | 파일 |
|------|-----------|------|
| Topic 1 | 85 | q_topic1.json |
| Topic 2 | 85 | q_topic2.json |
| Topic 3 | 90 | q_topic3.json |
| Topic 4 | 102 | q_topic4.json |
| Topic 5 | 85 | q_topic5.json |
| **합계** | **447** | |

### Step 4: 앱 UI 구현
- 단일 HTML 파일 (`mm-study.html`)
- 모바일 퍼스트, 한 문제씩 표시
- CSS 변수 기반 디자인 시스템
- localStorage로 진행률/오답 저장

**학습 모드:**
- 전체 학습 (순서대로)
- 토픽별 학습 (선택)
- 모의고사 (55문제 랜덤, 선다 85% + 주관 15%)
- 빈칸만 학습
- 오답 복습

### Step 5: 품질 검수 및 버그 수정

**발견된 문제:**
1. 빈칸 틀린 후 다음 빈칸 문제에서 입력폼이 빨간색 + disabled 상태 유지
2. 빈칸 정답 매칭이 너무 엄격 (순서/구두점 달라지면 오답 처리)
3. Topic 1 Q5 선지와 질문 불일치

**수정 내용:**
- `renderQuestion()`에서 `input.disabled = false`, `style.borderColor = ''`, `style.background = ''` 리셋 추가
- `matchBlank()` 함수 신규 구현: 공백/쉼표/괄호 무시, 토큰 순서 무관, 부분 포함 허용
- Q5 질문을 "분리" → "불일치와 분리"로 명확화

### Step 6: 출제예상도 분류 (병렬 에이전트)
- 4개 에이전트 동시 실행 (T1+T2, T3, T4, T5)
- 별점 기준:
  - 5: 반드시 출제 (PDF 빈칸, 핵심 정의/프레임워크)
  - 4: 출제 가능성 높음 (비교, 프로세스, 전략 적용)
  - 3: 출제 가능성 보통 (세부 내용, 보조 개념)
  - 2: 출제 가능성 낮음 → **삭제**
  - 1: 거의 출제 안됨 → **삭제**
- blank 문제는 무조건 별 5 (삭제 금지)

| 토픽 | 삭제 전 | 삭제 후 | 삭제수 |

| T2 | 85 | 77 | 8 |

### Step 7: 이어서 풀기 기능
- `saveSession()`: 나가기 시 현재 모드/진행 위치/답안/문제 ID 목록을 localStorage에 저장
- `resumeSession()`: 홈 화면에서 이전 세션 복원
- `quitQuiz()`: 세션 저장 후 홈으로 (vs. `endQuiz()`: 세션 삭제 후 결과 화면)
- 문제 이동(다음/건너뛰기) 시마다 자동 저장

### Step 8: 마크다운 정리본 작성
- `수업내용정리/마케팅관리_중간고사_정리.md`
- Topic 1~5 전체 내용 구조화
- 빈칸 전수 정리표 포함

### Step 9: PDF 빈칸 누락 검증 및 보완
- PDF 원문에서 `____` 패턴 전수 추출 (52개 위치)
- 문제은행과 1:1 대조
- Topic 1 다이어그램 빈칸 4개 누락 발견 → 추가
  - 마케팅믹스(4P), 마케팅 개념, 불확실성, CSR

## 6. 기술 스택

- **앱**: Vanilla HTML/CSS/JS (단일 파일, 의존성 없음)
- **PDF 파싱**: Python + PyMuPDF (fitz)
- **문제 생성**: Claude 병렬 에이전트 (토픽별 독립 실행)
- **데이터**: JSON (문제은행) → HTML 인라인 임베딩
- **저장**: localStorage (진행률/오답/세션)
- **배포**: GitHub Pages (정적 호스팅)
